## Features (by area)

### Analysis contract (`schemaVersion` 1.3.0)

| Area | What it is |
|------|------------|
| **Care navigation** | Optional `careNavigation`: `doToday`, `doThisWeek`, `beforeNextAppointment`, `whoToCall` (role, detail, phone hint, when to call). |
| **Medication reconciliation** | Optional `medicationReconciliation[]`: name, status enum, note — educational; confirm with a clinician. |
| **Confidence** | Chips driven from `overallSummaryConfidence` / `keyTakeawaysConfidence` where present. |

### APIs

| Route | Role |
|-------|------|
| **`POST /api/extract-preview`** | Server-side PDF text extraction; returns character count, quality (`empty` / `low` / `ok`), warnings (e.g. scanned/encrypted). Rate-limited. |
| **`POST /api/feedback`** | Accepts helpful/comment/metadata; logs safely; optional **`FEEDBACK_WEBHOOK_URL`** for outbound integration. |

### UI (results & upload)

| Piece | Role |
|-------|------|
| **CareNavigationPanel** | Renders care navigation; local-only phone notes (`localStorage`) so users can jot clinic numbers without sending them in feedback. |
| **MedicationReconciliationSection** | Status-colored cards for med reconciliation items. |
| **TrustSafetyIntegrationsPanel** | Safety, limitations, and handoff-oriented context. |
| **RedFlagQuiz** | Short yes/no “quick safety check.” |
| **SummaryConfidenceChips** | Visual confidence for summary/takeaways. |
| **FeedbackBar** | Submits to `/api/feedback`. |
| **Upload flow** | After file pick, debounced call to extract-preview with clear messaging if text is weak or missing. |
| **Results navigation** | In-page jump links for non-discharge documents; discharge sidebar shows “Care navigation” only when data exists (no dead anchors). |

### Documentation & config

| Artifact | Role |
|----------|------|
| **`OPERATIONS.md`** | High-level data flow, BAA reminders, feedback/PHI guidance, webhook env, schema versioning note. |
| **`.env.example`** | Documents **`FEEDBACK_WEBHOOK_URL`**. |

## Benefits

### For patients & caregivers

- **Clearer next steps** — Today / this week / before visit reduces overwhelm after a discharge or complex visit.
- **Safer mental model** — Confidence chips, red-flag quiz, and trust panel reinforce that output is educational, not a diagnosis.
- **Fewer wasted uploads** — PDF preview catches empty or low-text files before an expensive or confusing full analysis.
- **Actionable contacts** — “Who to call” plus optional local phone notes supports real-world follow-through.

### For operators & compliance-minded teams

- **Audit-friendly outputs** — Schema version and traceability fields support regression and incident review.
- **Feedback without building storage** — Log + optional webhook supports ticketing or analytics without committing PHI in comments.
- **Documented posture** — `OPERATIONS.md` gives a starting point for BAAs, vendors, and incident response.
- **Automated verification** — Single command line for tests, lint, types, and production build reduces regressions when changing prompts or schema.
